Задание:

Аналитикам международной компании, предоставляющей услуги реалтайм парсинга
сайтов, необходимо ручное добавление openapi в их api gateway. В связи с этим
требуется разработать микросервис для обработки openapi файлов (yml, json).
Сервис должен принимать файл и выводить тип метода, название метода, его краткое
описание. Далее аналитик указывает необходимые для работы методы из загруженных
и сохраняет под новым именем апи с выбранными методами. Загружать файлы могут
разные аналитики, но возможно в какой-то момент несогласованности сотрудников
система получит ранее загруженный файл.

Реализация на DjangoRestFramework, никаких шаблонов верстать не надо, все через
api. Сервис завернуть в докер, думаю с этим сложностей не возникнет.


----

Описание эндпоинтов

POST /files/ - загрузка нового файла
  предполагается, что будет передаваться multipart/form-data,
  поле schema_file (yaml/json, валидный OpenAPI) и опциональное поле name

POST /files/<name>/extract - создание нового файла на основе имеющегося (передаёт список операций и указание на конкретный файл)
  JSON формат тела запроса
  {"new_name": "new_schema_name", operation_ids: ["first", "second"]}

GET /files/<name> - получить данные файла, в том числе операции из файла
GET /files/<name>/file?output=json - получить файл в формате JSON
GET /files/<name>/file?output=yaml - получить файл в формате YAML
GET /files/ - получить список файлов
DELETE /files/<name> - удаление файла
